# UI/UX Style Guide: AI-Personal-OS

## 1. Visual Identity
- **Theme:** Dark Mode nativa (Deep Grays e Jet Black). Il bianco puro è vietato per ridurre l'affaticamento visivo.
- **Accenti:** Un solo colore primario (es. Electric Blue o Indigo) per le azioni principali e il branding.
- **Colori Semantici:** - Red: Criticità/Blocco.
  - Amber: Shadow Task in ritardo/At Risk.
  - Emerald: Deep Work completato/On Track.

## 2. Typography & Hierarchy
- **Font:** Sans-serif moderno e altamente leggibile (es. Inter o Geist).
- **Size Strategy:** Testo piccolo e denso per le liste task (per vedere tutto a colpo d'occhio), testo più ampio e arioso per il Daily Briefing dell'AI.

## 3. UI Patterns (Power User focus)
- **The "Command Bar" First:** L'interazione primaria deve essere una Command Bar (CMD+K) centrale.
- **Glassmorphism:** Uso leggero di trasparenze e blur per i pannelli dell'AI (per dare l'idea che l'IA sia uno strato "sopra" i dati deterministici).
- **Micro-interazioni:** Animazioni quasi impercettibili (sottili transizioni di stato) per non rallentare l'interfaccia.

## 4. AI-Specific UI
- **AI-Generated Content:** Deve avere un background leggermente diverso o un bordo sfumato per distinguerlo dai dati inseriti manualmente dall'utente.
- **Streaming UI:** Le risposte dell'AI devono essere visualizzate in streaming (caricamento progressivo) per ridurre la latenza percepita.

# Project Context: Personal AI-OS

## Vision
A Personal Assistant that merges deterministic task management (akin to Todoist) with the proactive reasoning capabilities of GenAI. The overarching goal is to dissolve the cognitive overload experienced by a Technical Lead / Architect.

## Core Problem
The user expertly manages their own tasks but depends heavily on external deliverables and outputs generated by others (Team, Stakeholders). The system must continuously and autonomously monitor the user's "Critical Path" in relation to strict external deadlines.

## Key Concepts
- **Shadow Tracking:** Active algorithmic monitoring of tasks formally assigned to others that tangentially or directly impact the user's workflow.
- **Deep Work Protection:** The AI must proactively reserve and defend contiguous time blocks (Deep Work Slots) dedicated to high-cognitive-load execution (architecture design, technical analysis), preventing fragmented meetings or minor administrative tasks from polluting focus time.
- **Context Awareness:** The assistant fundamentally understands the state of the project timeline; for example, if an Epic enters the "Release" phase, the priority of bug-fixing tasks must dynamically escalate without manual intervention.

## Architecture Decisions (Immutable)
- **Backend:** FastAPI with Asynchronous SQLModel (SQLite/PostgreSQL in production).
  - Leverages `asyncpg` for Postgres and `aiosqlite` for local dev.
- **AI Engine:** PydanticAI (Priority: Type safety and structured output strictly over flexibility).
- **Dependency Management:** All tasks must support `blocked_by` relationships to effectively calculate the Critical Path.
- **Frontend Stack:** React 18+ (Vite) with TypeScript, Zustand, and TanStack React Query.
- **Project Structure:** Clean Architecture (Model-Controller-Service pattern securely delegating business logic and DB access to isolated Services).
